# Прототип детекции состояний столика по видео

- детекция событий `table_empty`, `table_occupied`, `approach`;
- сохранение событий в `Pandas DataFrame` (экспорт в CSV);
- расчет базовой аналитики: средняя задержка между уходом гостя и подходом следующего;
- визуализация ROI (bounding box) на выходном видео `output.mp4`.

## Зависимости

```bash
pip install -r requirements.txt
```

## Запуск

Базовый запуск (ROI выбирается вручную через `cv2.selectROI`):

```bash
python main.py --video video1.mp4
```

Если в папке проекта лежит ровно нужный `.mp4`, можно запустить и без аргументов:

```bash
python main.py
```

Запуск с фиксированным ROI (без интерактивного выбора):

```bash
python main.py --video video1.mp4 --roi 540 300 240 180
```

Если окно выбора ROI в вашей среде не реагирует (часто бывает в IDE/Remote), используйте только запуск с `--roi`.

Полезные параметры:
- `--output output.mp4` путь выходного видео;
- `--events_csv events.csv` таблица событий;
- `--report_txt report.txt` текстовый отчет;
- `--display` показывает окно обработки в реальном времени.

## Что сохраняет скрипт

После выполнения формируются файлы:
- `output.mp4` видео с визуализацией состояния столика;
- `events.csv` все события с таймкодами;
- `delays.csv` пары `empty -> approach` и задержки;
- `report.txt` краткий отчет со средней задержкой.

## Логика детекции

Используется вычитание фона (`BackgroundSubtractorMOG2`) в OpenCV:
1. На каждом кадре выделяются движущиеся области.
2. Считается доля движения внутри ROI столика.
3. Если доля выше порога, стол считается занятым.
4. Переходы состояния стабилизируются во времени, чтобы убрать шум.
5. Событие `approach` фиксируется при переходе `empty -> occupied` после периода пустоты.
